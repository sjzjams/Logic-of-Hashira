# 原生 Android 层开发 PRD

> 版本：v1.0 | 创建日期：2026-06-05 | 状态：draft

---

## 1. 背景和问题

### 1.1 当前状态

Calorie Snap 项目已完成 Flutter/Dart 层的基础设施和功能体验层开发，包括：
- 8 幕 UI 流程的状态机实现
- 动画系统（Hero 转场、Interval 时序编排、Overlay 驱动动画等）
- 数据模型和营养计算逻辑
- 埋点系统

### 1.2 存在的问题

1. **原生相机能力不足**：当前使用 `camera` 插件（Camera2 包装），无法实现实时 YOLO 推理和帧分析
2. **不支持实时识别**：`SegmentationEngine.supportsRealtime` 返回 `false`，只能在拍照后异步执行分割
3. **缺少实时视觉反馈**：无法在预览时显示食物轮廓高亮，用户体验不符合 PRD 设计
4. **状态机未完整对接**：Dart 层状态机未与原生层完整对接，异常处理不完善

---

## 2. 目标和成功指标

### 2.1 主要目标

完成原生 Android 层（Kotlin/C++）开发，实现完整的 8 幕用户体验，包括：
1. 集成 CameraX，实现实时预览和帧捕获
2. 实现实时 YOLOv8-seg 推理
3. 在预览上显示食物轮廓高亮
4. 实现掩码缓存优化
5. 实现完整的 AI 状态机对接
6. 实现时间投票稳定器

### 2.2 成功指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 相机预览帧率 | ≥ 30 FPS | `adb shell dumpsys gfxinfo` |
| 实时推理帧率 | ≥ 15 FPS | `System.currentTimeMillis()` 计时 |
| 识别准确率 | ≥ 85% | 测试集验证 |
| 内存占用 | < 200 MB | Android Profiler |
| 电池消耗 | < 10%/小时 | Android Profiler |

## 3. 用户故事

### 3.1 用户故事 1：实时食物识别

**作为** 用户，  
**我想要** 在相机预览中实时看到食物识别结果，  
**以便** 我知道应用正在正确识别食物，并且可以及时拍照。

**验收标准**：
- 打开相机页面后，预览画面中实时显示食物轮廓高亮
- 高亮延迟 < 200ms
- 预览帧率 ≥ 30 FPS

### 3.2 用户故事 2：快速拍照识别

**作为** 用户，  
**我想要** 点击拍照按钮后快速得到识别结果，  
**以便** 我可以及时记录我的餐食。

**验收标准**：
- 点击拍照后 1 秒内显示识别结果
- 识别和展示过程有流畅的动画
- 可以保存识别结果到历史记录

### 3.3 用户故事 3：稳定的识别结果

**作为** 用户，  
**我想要** 识别结果稳定不抖动，  
**以便** 我可以信任应用的识别结果。

**验收标准**：
- 同一食物在连续帧中的识别结果一致
- 使用时间投票机制稳定结果
- 置信度显示准确

## 4. 技术方案概览

### 4.1 技术栈

- **Kotlin**：Android 原生代码
- **C++**：NCNN 推理引擎和 YOLOv8-seg 实现
- **CameraX**：相机预览和帧捕获
- **NCNN**：神经网络推理引擎

## 5. 功能需求

### 5.1 INF2 原生相机管线

**描述**：集成 CameraX，实现实时预览和帧捕获。

**功能点**：
1. 添加 CameraX 依赖
2. 实现 `CameraManager` 类，管理相机生命周期
3. 实现 `FrameAnalyzer` 类，分析每一帧
4. 将 CameraX 预览与 Flutter Texture 绑定

**验收标准**：
- 相机预览正常运行
- 能够捕获预览帧
- 帧分析流程打通
- 支持前后摄像头切换

### 5.2 INF3 掩码缓存

**描述**：优化掩码存储和复用机制。

**功能点**：
1. 实现 `MaskCache` 类，管理掩码缓存
2. 使用 LRU 策略管理缓存大小
3. 支持掩码的快速查询和复用
4. 优化掩码存储格式

**验收标准**：
- 掩码缓存功能正常
- 缓存命中率 > 50%
- 磁盘占用优化
- 内存占用合理

### 5.3 INF4 AI 状态机

**描述**：实现完整的 8 幕状态机原生层对接。

**功能点**：
1. 在原生层实现状态机
2. 通过 MethodChannel 与 Dart 层同步状态
3. 实现状态转换的逻辑
4. 处理异常状态和恢复

**验收标准**：
- 状态机正常运行
- 状态同步准确
- 异常处理完善
- 状态转换有日志

### 5.4 INF5 时间投票稳定器

**描述**：通过 temporal voting 机制稳定识别结果。

**功能点**：
1. 实现 `TemporalVoter` 类，管理时间序列上的识别结果
2. 使用投票机制稳定识别结果
3. 实现置信度衰减和刷新机制
4. 优化投票窗口大小

**验收标准**：
- 识别结果稳定
- 置信度合理
- 响应延迟低（< 500ms）
- 支持动态窗口大小调整

### 5.5 T7 实时 YOLO 高亮

**描述**：在预览上显示食物轮廓高亮。

**功能点**：
1. 在 C++ 层实现实时 YOLO 推理
2. 将推理结果（检测框、掩码）传回 Dart 层
3. 在 Flutter UI 上绘制高亮轮廓
4. 优化渲染性能

**验收标准**：
- 实时 YOLO 推理正常运行
- 高亮轮廓正确显示
- 性能满足要求（≥ 15 FPS）
- 高亮延迟 < 200ms

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 相机预览帧率 | ≥ 30 FPS |
| 实时推理帧率 | ≥ 15 FPS |
| 内存占用 | < 200 MB |
| 电池消耗 | < 10%/小时 |
| 启动时间 | < 2 秒 |

### 6.2 兼容性需求

- 支持 Android 7.0（API 24）及以上
- 支持 armeabi-v7a 和 arm64-v8a ABI
- 支持前后摄像头
- 支持多种屏幕分辨率

### 6.3 安全需求

- 不收集用户隐私数据
- 相机权限使用时请求
- 图片数据仅本地处理，不上传服务器
